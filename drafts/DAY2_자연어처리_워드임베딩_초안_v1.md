## 2. 워드 임베딩이란 무엇인가 🟦

### 2.1 왜 필요한가

파이썬을 포함한 프로그래밍 언어는 **텍스트 자체를 계산하지 못합니다.** "사과"와 "바나나"가 얼마나 비슷한지 빼거나 더할 수 없죠. 그래서 단어를 **숫자 벡터(좌표)**로 바꿔야 합니다. 이 과정이 **워드 임베딩(Word Embedding)**입니다.

> 🟦 워드 임베딩 = "단어의 의미를 **여러 축(차원)을 가진 벡터 공간**에 좌표로 표현하는 기술". 단순히 번호를 매기는 게 아니라, 분석할 수 있는 구조를 만드는 것이 목적입니다(p3).

### 2.2 가장 단순한 방법 — 원핫 인코딩

**원핫 인코딩(One-Hot Encoding)**은 단어마다 고유 번호를 주고, 그 자리만 1이고 나머지는 0인 벡터로 표현합니다.

```python
import numpy as np

words = ["밥", "김치", "배추", "국"]
one_hot = np.eye(len(words), dtype=int)   # 단위행렬: 대각선만 1

for w, vec in zip(words, one_hot):
    print(w, "->", vec.tolist())
```

**실제 출력** (🟩 직접 실행):
```
밥 -> [1, 0, 0, 0]
김치 -> [0, 1, 0, 0]
배추 -> [0, 0, 1, 0]
국 -> [0, 0, 0, 1]
```

> 🟦 **장점**: 직관적이고 쉽습니다. 지역 코드처럼 종류가 적고 단순한 범주를 숫자로 바꿀 때 적합합니다(p3).

**한계 두 가지** (🟦 p3):

1. **차원 폭발** — 단어가 1만 개면 벡터 길이도 1만. 단어가 늘수록 벡터가 무한정 커져 대용량에 비효율적입니다.
2. **의미 관계 표현 불가** — 모든 단어 사이 거리가 똑같습니다. "김치"와 "배추"가 가깝고 "밥"과는 덜 가깝다는 정보를 전혀 담지 못합니다.

> 🟩 왜 거리가 모두 같을까요? 어떤 두 원핫 벡터를 골라도 서로 다른 자리에 1이 하나씩 있으므로, 코사인 유사도(아래 3장)가 항상 0이 됩니다. 즉 컴퓨터 입장에선 **모든 단어가 똑같이 무관**합니다.

### 2.3 더 똑똑한 방법 — 워드 벡터(밀집 표현)

이 한계를 푸는 것이 **워드 벡터(Word Vector)**입니다. 대표적으로 **Word2Vec**이 있습니다(p4).

| 구분 | 원핫 인코딩 | 워드 벡터(Word2Vec 등) |
|---|---|---|
| 벡터 길이 | 단어 수만큼 늘어남 | **고정 차원**(예: 100~200) |
| 의미 관계 | 표현 못 함 | 비슷한 단어가 **가까이** 모임 |
| 값 | 대부분 0 (희소) | 실수로 꽉 참 (밀집) |
| 강점 | 작고 단순한 데이터 | **대규모 데이터일수록 강력** |

> 🟦 워드 벡터는 새 단어가 추가돼도 차원이 늘지 않고, **비슷한 의미의 단어가 벡터 공간에서 자연스럽게 가까이 모입니다.** 오늘날 "워드 임베딩"이라고 하면 보통 이 방식을 뜻합니다(p4).

## 3. 단어의 의미를 거리로 측정하기 🟦

### 3.1 코사인 유사도 — 왜 "각도"인가

두 단어 벡터가 얼마나 비슷한지는 **코사인 유사도(Cosine Similarity)**, 즉 두 벡터가 이루는 **각도**로 잽니다.

| 두 벡터의 각도 | 코사인 값 | 의미 | 예시(🟦) |
|---|---|---|---|
| 0° (같은 방향) | **+1** | 매우 유사 | 좋다–훌륭하다 |
| 90° (수직) | **0** | 서로 무관 | 사과–컴퓨터 |
| 180° (반대 방향) | **−1** | 정반대 의미 | 좋다–나쁘다 |

> 🟩 길이가 아니라 **방향(각도)**을 보는 이유: 문서가 길어 단어가 많이 나오면 벡터가 길어지지만, "어떤 주제를 향하는가"라는 방향은 길이와 무관하게 유지됩니다. 그래서 텍스트 유사도에는 거리보다 각도가 자주 쓰입니다.

`scikit-learn`으로 직접 계산해 봅니다(코퍼스는 4.4에서 만든 TF-IDF 벡터를 사용 — 아래 4장 코드를 먼저 실행한 상태라고 가정).

```python
from sklearn.metrics.pairwise import cosine_similarity

# X = 4개 리뷰의 TF-IDF 행렬 (4장에서 생성)
sim = cosine_similarity(X)
print(sim.round(3))
```

**실제 출력** (🟩 직접 실행):
```
[[1.    0.322 0.132 0.   ]
 [0.322 1.    0.    0.161]
 [0.132 0.    1.    0.396]
 [0.    0.    0.396 1.   ]]
```

해석: doc0("배송 빠르다…")과 doc1("배송 느리다…")은 **0.322**로 서로 가장 비슷합니다(둘 다 배송 이야기). doc2·doc3(가격·품질 이야기)은 **0.396**으로 묶입니다. 반면 doc0–doc3은 **0.0** — 공통 단어가 없어 완전히 무관합니다.

### 3.2 워드 벡터의 핵심 원리 — 분산 가설

워드 벡터가 작동하는 근거는 한 문장으로 요약됩니다.

> 🟦 **분산 가설(Distributional Hypothesis)**: "비슷한 맥락에서 등장하는 단어들은 비슷한 의미를 가진다."(p5)

즉 "환불"과 "취소"가 늘 비슷한 문장에서 쓰인다면, 모델은 둘을 가까운 좌표에 놓습니다. 더 놀라운 것은 **벡터 연산으로 의미 관계를 추론**할 수 있다는 점입니다.

```
왕 - 남자 + 여자 ≈ 여왕      (🟦 p5)
King - Man + Woman ≈ Queen   (🟦 p10)
```

> 🟩 **주의 — 이것은 "근사" 결과입니다.** 실제로는 등호(=)가 아니라, 계산한 벡터에 **가장 가까운 단어를 찾는(최근접 탐색)** 것이며, 충분히 크고 잘 학습된 모델에서만 깔끔하게 나타납니다. 작은 데이터로 학습하면 엉뚱한 단어가 나오기도 합니다.

> 🟦 ChatGPT 같은 모델도 결국 말을 **벡터 공간으로 바꿔 의미를 숫자로 다룬 뒤** 다시 자연스러운 언어로 되돌리는 과정을 거칩니다(p5). 사람은 "약간 비슷하다"처럼 모호하게 말하지만, 컴퓨터는 "0.82"처럼 정량적으로 계산합니다.

### 3.3 좋은 임베딩을 위한 고려사항 🟦

강의가 짚는 4가지(p5):

1. **데이터 품질·도메인** — 충분히 많고 질 좋은 텍스트가 필요하며, 특정 분야 데이터로 학습하면 그 분야 성능이 올라갑니다.
2. **차원 수 설정** — 너무 높으면 과적합·계산 비용↑, 너무 낮으면 서로 다른 단어가 같은 좌표에 뭉쳐 의미가 왜곡됩니다.
3. **데이터 균형** — 한쪽으로 치우친 데이터는 편향된 임베딩을 만듭니다(전처리 품질이 중요).
4. **검증·업데이트** — 여러 방법을 비교 실험하고, "개이득"처럼 새로 생기거나 뜻이 변하는 말에 맞춰 주기적으로 다시 학습해야 합니다.

## 4. 빈도 분석과 TF-IDF 🟦

### 4.1 가장 기초 — 단어 빈도 세기

텍스트를 수치화하는 가장 단순한 방법은 **단어가 몇 번 나왔는지 세는 것**입니다(p6).

- **단순 빈도**: 긍정·부정 단어가 몇 번 나왔는지 세어 점수화.
- **상대 빈도**: 전체 평균과 비교해 "이 글이 유독 강한 표현을 쓰는가"를 봄.
- **이진 빈도(Binary)**: 등장 여부만 예/아니오로 — 빠르지만 의미 파악엔 한계.

> 🟦 **한계**: 단순 빈도만으로는 부정 표현이 드문 글이 모두 긍정으로 잘못 해석될 수 있습니다(p6).

### 4.2 TF-IDF는 왜 나왔나

> 🟦 **"빈도가 높다 = 글을 대표한다?"** 이 질문에서 TF-IDF가 출발했습니다. 초등학생 일기에 "나는 오늘"이 가장 많이 나온다고 해서 그게 일기의 핵심 주제는 아니니까요(p6).

자주 나오는 흔한 단어에는 **벌점**을 주고, 특정 문서에만 콕 집어 나오는 단어를 **핵심 키워드**로 끌어올리자는 발상입니다.

> 🟦 강의는 이 아이디어를 **1958년 IBM의 한스 피터 룬(Hans Peter Luhn)**이 제안했다고 소개합니다(p6).
>
> 🟩 **역사 보충(정확성)**: 룬은 *단어 빈도 기반 자동 색인(TF 쪽)*을 개척한 인물이 맞습니다. 다만 TF-IDF의 핵심인 **IDF(역문서 빈도)**는 보통 **1972년 카렌 스파크 존스(Karen Spärck Jones)**가 제안한 것으로 봅니다. 즉 "TF-IDF = 룬 혼자, 1958"보다는 두 사람의 아이디어가 결합된 것으로 이해하면 더 정확합니다.

### 4.3 원리와 공식

- **TF(Term Frequency, 단어 빈도)**: 한 문서 안에서 그 단어가 얼마나 자주 나오는가.
- **IDF(Inverse Document Frequency, 역문서 빈도)**: 전체 문서 중 그 단어가 **얼마나 드물게** 나오는가.

> 🟦 핵심: **특정 문서에선 자주 나오지만 전체에선 희귀한 단어**에 높은 가중치를 줍니다. IT 문서 모음에서 "컴퓨터"는 흔해서 중요도가 낮고, "블록체인"처럼 일부 문서에만 몰린 단어가 핵심이 됩니다(p7).

공식은 다음과 같습니다(🟦 p7).

```
w(x, y) = tf(x, y) × log( N / df(x) )
```

- `tf(x, y)` : 문서 y에서 단어 x의 빈도
- `N` : 전체 문서 수
- `df(x)` : 단어 x가 등장한 문서 수
- `log` : 흔한 단어의 가중치를 완만하게 깎는 역할

> 🟩 직관: 단어가 **모든 문서에 다 나오면** `N/df(x)`가 1에 가까워지고 `log(1)=0` → 가중치가 0으로 깎입니다. 그래서 the, 이, 가 같은 흔한 말은 자동으로 약해집니다(불용어 자동 감소 효과).

### 4.4 scikit-learn으로 TF-IDF 실습 🟩

DAY1에서 전처리(불용어 제거)를 끝내 **내용어만 남은 리뷰 4개**를 코퍼스로 씁니다. (이 코퍼스를 4·5·3장에서 공통으로 사용합니다.)

```python
from sklearn.feature_extraction.text import TfidfVectorizer

corpus = [
    "배송 빠르다 포장 꼼꼼 만족",   # doc0: 배송 긍정
    "배송 느리다 포장 엉망 실망",   # doc1: 배송 부정
    "가격 저렴 가격 합리 품질 만족", # doc2: 가격 긍정
    "가격 비싸다 품질 별로 실망",   # doc3: 가격 부정
]

# 한국어는 기본 token_pattern이 한 글자 단어를 버리므로 아래 패턴으로 지정
vec = TfidfVectorizer(token_pattern=r"(?u)\b\w+\b")
X = vec.fit_transform(corpus)

print("행렬 모양 (문서 수, 단어 수):", X.shape)
print("단어 사전:", vec.get_feature_names_out().tolist())
```

**실제 출력** (🟩 직접 실행):
```
행렬 모양 (문서 수, 단어 수): (4, 14)
단어 사전: ['가격', '꼼꼼', '느리다', '만족', '배송', '별로', '비싸다', '빠르다', '실망', '엉망', '저렴', '포장', '품질', '합리']
```

각 문서가 14차원 TF-IDF 벡터로 바뀌었습니다. doc0에서 점수가 높은 단어를 봅니다.

```python
import numpy as np
arr = X.toarray()
fn = vec.get_feature_names_out()
top = [(fn[i], round(float(arr[0][i]), 3)) for i in arr[0].argsort()[::-1][:5]]
print("doc0 상위 TF-IDF:", top)
```

**실제 출력** (🟩 직접 실행):
```
doc0 상위 TF-IDF: [('빠르다', 0.509), ('꼼꼼', 0.509), ('포장', 0.401), ('배송', 0.401), ('만족', 0.401)]
```

doc0에만 등장한 "빠르다·꼼꼼"이 가장 높은 점수를 받았습니다. 여러 문서에 두루 나온 "배송·만족"은 상대적으로 낮습니다 — IDF 벌점이 작동한 결과입니다.

**TF-IDF로 본 "분포 가설의 가장 단순한 형태"** — 같은 문서에 함께 등장한 단어끼리 유사하게 보입니다.

```python
from sklearn.metrics.pairwise import cosine_similarity

word_vectors = X.T.toarray()          # 행=단어, 열=문서 (단어를 문서로 표현)
def word_sim(a, b):
    ia, ib = list(fn).index(a), list(fn).index(b)
    return round(float(cosine_similarity([word_vectors[ia]], [word_vectors[ib]])[0, 0]), 3)

print("가격-품질:", word_sim("가격", "품질"))   # 같은 문서에 자주 함께 등장
print("배송-가격:", word_sim("배송", "가격"))   # 함께 등장한 적 없음
```

**실제 출력** (🟩 직접 실행):
```
가격-품질: 0.944
배송-가격: 0.0
```

> 🟩 **주의 — 이건 Word2Vec이 아닙니다.** 위는 "같은 문서에 함께 나왔는가(공기 빈도)"만 본 **가장 단순한** 유사도입니다. Word2Vec(6장)은 더 나아가 *주변 단어의 패턴*까지 학습해, 한 번도 같은 문서에 없던 단어라도 비슷하면 가깝게 놓을 수 있습니다.

### 4.5 TF-IDF의 장점과 한계 🟦

- **장점**: 해석이 명확·직관적, 불용어 자동 감소, 문서 유사도 계산에 강함(p7).
- **한계**: **단어 순서와 문맥을 무시**합니다. "안 좋다"와 "좋다"의 미묘한 차이를 구별하기 어렵습니다(p7).
- **보완**: 품사 태깅을 결합하거나 **Word2Vec·BERT** 같은 문맥 임베딩과 섞어 한계를 극복합니다(p7).

## 5. 토픽 모델링 — 문서 속 숨은 주제 찾기 🟦

### 5.1 워드 임베딩 기술의 발전 과정

> 🟦 강의가 정리한 흐름(p8):

| 시기 | 기술 | 핵심 |
|---|---|---|
| 1970년대 | 빈도 기반 | 단어가 얼마나 자주 나오는지로 배열 |
| 1990년대 | **LSA** | 좌표 평면에 배치해 비슷한 단어를 주제로 묶기 시작 |
| 2000년대 | **LDA** | 본격적인 토픽 모델링 등장 |
| 2013년 | **Word2Vec·GloVe** | 단어를 벡터로 변환(신경망 임베딩) |
| 2018년 | **BERT·GPT** | 스스로 문맥을 이해하는 모델 |
| 2020년~ | 추론 기반 | 요약·복잡한 대화가 가능한 모델 |

> 🟩 연도 보충: Word2Vec은 2013년 구글, **GloVe는 2014년 스탠퍼드**에서 나왔습니다. 큰 흐름을 잡는 용도로 보면 됩니다.

### 5.2 토픽 모델링이란

> 🟦 **토픽 모델링**: 비슷한 위치에 모인 단어·문서를 같은 **주제(Topic)**로 묶는 기술. 단순 긍/부정 분류를 넘어, 수천 개 고객 리뷰에서 **배송·품질·가격·서비스** 같은 주제를 자동으로 뽑아냅니다. 대표적인 **비지도 학습**입니다(p8).

### 5.3 LDA 실습 🟩

대표 알고리즘 **LDA(Latent Dirichlet Allocation)**를 4.4의 같은 리뷰 코퍼스에 적용합니다. 주제 2개를 찾아보겠습니다.

```python
from sklearn.feature_extraction.text import CountVectorizer
from sklearn.decomposition import LatentDirichletAllocation

cv = CountVectorizer(token_pattern=r"(?u)\b\w+\b")
counts = cv.fit_transform(corpus)          # LDA는 정수 빈도(BoW)를 입력으로 씀
terms = cv.get_feature_names_out()

lda = LatentDirichletAllocation(n_components=2, random_state=42)  # 재현성 위해 시드 고정
doc_topic = lda.fit_transform(counts)

for k, comp in enumerate(lda.components_):
    top = [terms[i] for i in comp.argsort()[::-1][:4]]
    print(f"Topic {k}:", top)

print("문서별 주제 분포:")
print(doc_topic.round(3))
```

**실제 출력** (🟩 직접 실행):
```
Topic 0: ['가격', '품질', '만족', '합리']
Topic 1: ['포장', '배송', '실망', '엉망']
문서별 주제 분포:
[[0.095 0.905]
 [0.09  0.91 ]
 [0.924 0.076]
 [0.907 0.093]]
```

해석: 알고리즘이 라벨 없이도 **Topic 0 = 가격/품질**, **Topic 1 = 배송/포장** 두 주제를 스스로 찾았습니다. doc0·doc1은 Topic 1(배송)에 0.9 이상, doc2·doc3은 Topic 0(가격)에 쏠려 있습니다.

> 🟩 **주의**: 주제 번호(0, 1)는 **순서나 우열이 아니라 임의의 이름표**입니다. 실행 때마다(시드가 다르면) 번호가 바뀔 수 있고, "Topic 0이 무슨 주제인가"는 결국 **사람이 상위 단어를 보고 해석**해야 합니다.

### 5.4 결과를 어떻게 보나 🟦

> 🟦 LDA 분석은 보통 **문서-주제 분포 → 주제-단어 분포 → 주제 선택 → 단어 생성**의 4단계로 진행되며, 결과는 "주제별로 어떤 단어가 어떤 비율로 쓰였는지"를 키워드 묶음으로 보여 줍니다(p9).
>
> **주제 간 거리 맵(Intertopic Distance)**: 주제들 사이 유사성을 2D 평면에 표시해, 가까울수록 비슷한 내용을 다룹니다. 축(차원)이 너무 많아 주제가 잘 안 나뉘면 **차원 축소**로 조정합니다(p9). 마지막엔 사람이 직접 검토·보정하는 과정이 필요합니다.

## 6. Word2Vec — 의미를 담는 밀집 벡터 🟦

### 6.1 학습 원리

> 🟦 **Word2Vec**('Word to Vector')은 단어를 고정 차원의 **밀집 벡터**로 바꾸는 기술로, **분포 가설**을 핵심으로 합니다(p10).
> - 문맥 창(**Window**) 크기를 보통 **5~20개 단어**로 잡아 주변 단어로부터 학습합니다.
> - 긍정/부정, 시점, 성별 같은 의미 속성과 **방향성**을 벡터에 반영합니다.
> - 그래서 "환불"과 "취소"를 (원핫과 달리) 비슷한 맥락으로 묶을 수 있습니다.

### 6.2 gensim으로 Word2Vec 다루기 🟩

Word2Vec은 `scikit-learn`이 아니라 **gensim** 라이브러리를 표준으로 씁니다(여기서 라이브러리가 바뀝니다).

```bash
python -m pip install gensim
```

```python
from gensim.models import Word2Vec

# 입력: 문장을 토큰 리스트로 나눈 형태 (실제로는 대규모 코퍼스가 필요)
sentences = [
    ["배송", "빠르다", "포장", "꼼꼼", "만족"],
    ["배송", "느리다", "포장", "엉망", "실망"],
    ["가격", "저렴", "합리", "품질", "만족"],
    ["가격", "비싸다", "품질", "별로", "실망"],
]

model = Word2Vec(
    sentences,
    vector_size=100,   # 임베딩 차원
    window=5,          # 문맥 창 크기
    min_count=1,       # 최소 등장 횟수
    sg=1,              # 1=Skip-gram, 0=CBOW
    seed=42, workers=1 # 재현성
)

print(model.wv["가격"].shape)              # (100,) : 100차원 벡터
print(model.wv.most_similar("가격"))        # '가격'과 가까운 단어들
print(model.wv.similarity("가격", "품질"))   # 두 단어 코사인 유사도
# 벡터 연산(유추): 왕 - 남자 + 여자 ≈ ?
# print(model.wv.most_similar(positive=["여자", "왕"], negative=["남자"]))
```

> 🟩 **출력을 싣지 않은 이유**: 이 글을 쓴 환경에는 `gensim`이 설치되어 있지 않아 **실제 출력을 검증할 수 없었습니다.** 또한 위처럼 **문장 4개짜리 작은 데이터로는 의미 있는 유사도가 나오지 않습니다.** Word2Vec의 진가(왕−남자+여자≈여왕 같은 결과)는 **수십만 문장 이상의 대규모 코퍼스**에서만 드러납니다. 직접 실행할 때는 큰 코퍼스나 **사전 학습된 모델**을 사용하세요(임의의 결과값을 지어내지 않았습니다).

### 6.3 고차원 벡터를 눈으로 보기 — PCA / t-SNE

> 🟦 Word2Vec 벡터는 보통 **100~200차원 이상**이라 그대로는 그릴 수 없습니다. 그래서 **차원을 2D로 줄여** 단어들의 군집 구조를 눈으로 봅니다(p10~11).

| 기법 | 특징 | 언제 |
|---|---|---|
| **PCA** | 선형, 빠르고 전체 분산 보존 | 큰 줄기·단순 비교 |
| **t-SNE** | 비선형, 국소 군집을 잘 드러냄 | 복잡한 비선형 군집 구조 |

아래는 어떤 벡터든 2D로 줄여 그리는 코드입니다(4.4의 TF-IDF 단어 벡터로 실행 가능).

```python
from sklearn.decomposition import PCA
import matplotlib.pyplot as plt

# word_vectors, fn = 4.4에서 만든 단어 벡터와 단어 이름
coords = PCA(n_components=2, random_state=42).fit_transform(word_vectors)

plt.figure(figsize=(6, 5))
plt.scatter(coords[:, 0], coords[:, 1])
for (x, y), name in zip(coords, fn):
    plt.annotate(name, (x, y))
plt.title("단어 벡터 2D 시각화 (PCA)")
plt.show()
```

> 🟩 한글 라벨이 네모(□)로 깨지면 `matplotlib` 폰트를 지정하세요: `plt.rcParams['font.family'] = 'Malgun Gothic'`(윈도우).
> 🟩 **t-SNE 주의**: `TSNE(perplexity=...)`의 `perplexity`는 **표본 수보다 작아야** 합니다(작은 데이터에서 에러의 흔한 원인).

> 참고로 위 PCA를 4.4의 단어 벡터에 적용하면 두 축이 설명하는 분산 비율은 `[0.486, 0.318]`로, 두 축이 전체 정보의 약 80%를 담습니다(🟩 직접 실행).

### 6.4 유사도 측정과 실무 활용 🟦

**거리 기반**(유클리드·맨해튼 거리)과 **코사인 유사도**를 함께 씁니다.

> 🟦 강의는 코사인 유사도가 **0.7 이상이면 높은 유사성**으로 본다고 안내합니다(p11).
>
> 🟩 단, **0.7은 절대 기준이 아니라 도메인에 따라 조정하는 경험적 임계값**입니다. 데이터·목적에 맞게 실험으로 정하세요.

**실무 활용 사례** (🟦 p11~12):

- **고객 문의 자동 배정** — 문의 텍스트를 벡터화해 기존 카테고리와 코사인 유사도를 재고, 0.7 이상이면 담당자에게 자동 배정.
- **상품 추천** — "남성용 운동화 − 남성 + 여성 ≈ 여성 운동화", "아이폰 − 스마트폰 + 태블릿 ≈ 아이패드"처럼 벡터 연산과 구매 패턴을 조합.
- **검색어 확장** — '빠른·당일·신속'처럼 가까운 단어를 자동으로 묶어 검색 범위를 넓힘.
- **기계 번역·문서 분류** — 문맥에 맞는 단어를 골라 번역하거나, 공통 키워드 패턴으로 문서를 자동 분류.

## 7. 세 가지 방법 비교 🟩

지금까지 배운 표현 방법을 한눈에 정리합니다.

| 기준 | 원핫 인코딩 | TF-IDF | Word2Vec |
|---|---|---|---|
| 형태 | 희소(0이 대부분) | 희소(가중치) | 밀집(실수) |
| 차원 | 단어 수만큼 | 단어 수만큼 | 고정(예: 100) |
| 의미 관계 | 없음 | 공기 빈도 수준 | **단어 의미·유추 가능** |
| 문맥/순서 | 무시 | 무시 | 주변 단어 반영 |
| 필요 데이터 | 적어도 됨 | 중간 | **대규모일수록 유리** |
| 대표 도구 | numpy | scikit-learn | gensim |
| 언제 | 범주가 적고 단순 | 키워드 추출·문서 유사도 | 의미 기반 추천·유사어 |

> 🟩 실무에서는 **TF-IDF로 빠르게 시작**해 보고, 의미·유추가 중요하면 **Word2Vec(또는 BERT 같은 문맥 임베딩)**으로 넘어가는 흐름이 흔합니다.

## 8. 자주 하는 실수

- **원핫으로 의미 유사도를 기대한다** → 원핫은 모든 단어 거리가 같아 유사도가 항상 0입니다. 의미가 필요하면 TF-IDF나 Word2Vec을 쓰세요.
- 🟩 **한국어 TF-IDF에서 한 글자 단어가 사라진다** → `TfidfVectorizer`의 기본 `token_pattern`이 1글자를 버립니다. `token_pattern=r"(?u)\b\w+\b"`로 지정하세요.
- 🟩 **작은 데이터로 Word2Vec을 학습하고 "왕−남자+여자=여왕"을 기대한다** → 그 결과는 대규모 코퍼스에서만 나옵니다. 작은 데이터엔 사전 학습 모델을 쓰세요.
- 🟩 **LDA 주제 번호를 우열로 해석한다** → 번호는 임의의 이름표일 뿐, 상위 단어를 보고 사람이 의미를 붙여야 합니다.
- 🟩 **코사인 0.7을 절대 기준으로 삼는다** → 도메인마다 다른 경험적 임계값입니다.
- 🟩 **t-SNE에서 `perplexity`가 표본 수 이상** → 에러가 납니다. 표본 수보다 작게 설정하세요.
- 🟩 **TF-IDF/원핫에 문맥을 기대한다** → 둘 다 단어 순서를 무시합니다. "안 좋다"의 부정을 잡으려면 문맥 임베딩이 필요합니다.

## 9. DAY2 핵심 정리

```
워드 임베딩 = 단어를 숫자 벡터로 (컴퓨터는 숫자만 계산)

표현 방법
  - 원핫    : 쉽지만 차원 폭발 + 의미 관계 없음
  - 워드벡터 : 고정 차원, 비슷한 단어가 가까이 (분산 가설)

유사도
  - 코사인 유사도 = 두 벡터의 각도 (0°→+1, 90°→0, 180°→−1)
  - 길이가 아니라 '방향'을 본다

빈도 기반
  - TF-IDF = 자주 + 희귀한 단어에 높은 점수 (w = tf × log(N/df))
  - 장점: 직관적·문서 유사도 / 한계: 문맥·순서 무시

토픽 모델링
  - LDA로 문서 속 주제를 비지도로 추출
  - 주제 번호는 임의, 상위 단어로 사람이 해석

Word2Vec
  - 분포 가설 기반 밀집 벡터, 의미 유추(왕−남자+여자≈여왕, 근사)
  - 고차원 → PCA(선형)/t-SNE(비선형)로 시각화
  - 실무: 추천·문의 배정·검색 확장·번역/분류

도구: 원핫=numpy / TF-IDF·LDA·PCA=scikit-learn / Word2Vec=gensim
```
