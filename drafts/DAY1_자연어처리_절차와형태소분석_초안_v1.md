## 1. 이번 DAY에서 배우는 것

이번 글은 **"사람의 언어(자연어)를 컴퓨터가 분석할 수 있게 다듬는 전체 과정"**을 처음부터 끝까지 따라갑니다. 모델을 학습시키는 단계까지 가지 않고, 그 **앞단(데이터 준비)**에 집중합니다.

- 텍스트 전처리 5단계: **토큰화 · 정제 · 정규화 · 불용어 제거 · 표제어/형태소**

## 2. 자연어 데이터란 무엇인가 🟦

### 2.1 왜 먼저 데이터의 종류부터 보는가

자연어 분석이 어려운 이유를 이해하려면, 우리가 익숙한 **정형 데이터**와 비교하는 것이 가장 빠릅니다.

| 구분 | 정형 데이터(Structured) | 비정형 데이터(자연어) |
|---|---|---|
| 구조 | 행·열로 정해진 표 형태 | 일정한 구조 없음 |
| 예시 | 데이터베이스, 스프레드시트(직원 ID·이름·나이) | 텍스트, 이미지, 음성, 영상 |
| 처리 도구 | Excel, SQL, 통계 분석 | 자연어 분석, 머신러닝/딥러닝 |
| 복잡성 | 상대적으로 단순 | 복잡함 |
| 분석 난이도 | 쉬움 | 어려움 |

> 🟦 강의 자료의 비교표(p5)를 정리한 것입니다.

정형 데이터는 "전화번호는 숫자처럼 보여도 계산 대상이 아니고, 나이는 계산·비교가 가능하다"처럼 **각 칸의 의미가 명확**합니다. 반면 자연어는 정해진 칸이 없고, 형태(문장·대화·시·소설)도 제각각이며, 양도 방대합니다.

### 2.2 자연어는 계층 구조를 가진다

자연어 데이터는 다음과 같은 **계층 구조**로 이루어져 있습니다.

```
글(문서) → 단락(Paragraph) → 문장(Sentence) → 단어(Word) → 형태소(Morpheme)
```

그래서 분석할 때 단어 하나만 보는 게 아니라 **문장과 문서 전체의 맥락**까지 같이 봐야 합니다. 이 글의 후반부에서 다루는 "형태소"는 이 사다리의 **가장 작은 의미 단위**입니다.

### 2.3 자연어가 어려운 4가지 이유

강의에서 강조하는 자연어의 고유 특징입니다. **이것이 바로 전처리가 필요한 이유**이기도 합니다.

1. **중의성(모호함)** — 같은 표현이 한 가지 의미로 고정되지 않습니다.
   예) "그가 그린 그림" → "그"가 누구인지는 문맥을 봐야 압니다.
2. **오탈자·약어** — "ㅋㅋㅋ", "잼써요"처럼 맞춤법을 벗어난 표현이 흔합니다.
3. **이모티콘·기호** — 👍 😊 처럼 글자가 아닌 감정 정보를 담습니다.
4. **문맥 의존성** — 같은 단어가 상황에 따라 뜻이 정반대가 됩니다.
   예) "우와, 대박!"(긍정) vs "대박 실수다…"(부정)

또한 어미 하나로 의미가 바뀝니다. "밥을 먹습니다."(서술)와 "밥을 먹습니까?"(의문)는 단어는 거의 같지만 문장의 성격이 다릅니다.

> 🟩 **여기서 기억할 한 가지**: 자연어가 어려운 이유(중의성·문맥)는 곧 "전처리에서 무엇을 함부로 지우면 안 되는가"로 이어집니다. 뒤의 [7장 자주 하는 실수](#9-자주-하는-실수)에서 다시 만납니다.

## 3. 자연어 처리(NLP)와 전처리의 큰 그림 🟦

### 3.1 NLP란?

**NLP(Natural Language Processing, 자연어 처리)**는 컴퓨터가 사람의 언어를 이해하고 처리하도록 만드는 기술 분야입니다. 번역, 챗봇, 음성 비서, 검색 엔진, 감성 분석, 문서 분류, 그리고 ChatGPT 같은 LLM이 모두 NLP의 대표 사례입니다.

핵심 전제는 한 문장으로 요약됩니다.

> **컴퓨터는 문자 그 자체를 이해하지 못한다. 그래서 자연어를 "처리 가능한 형태"로 바꿔 주어야 한다.**

### 3.2 LLM을 위한 데이터 준비 8단계

강의는 LLM 학습용 데이터를 만드는 과정을 다음 8단계로 제시합니다(p15).

```
1. 데이터 수집   (뉴스·블로그·책·논문·웹·위키)
2. 데이터 정제   (HTML 태그·광고·중복·깨진 문자 제거)
3. 문장 분리     (긴 글 → 문장 단위)
4. 토큰화        (문장 → 작은 단위 '토큰')
5. 형태소 분석   (토큰 → 의미 단위 + 품사)
6. 벡터화        (단어 → 숫자 벡터,  예: 사과 → [0.2, 0.7, 0.5])
7. 모델 학습
8. 모델 평가
```

> 🟩 이번 DAY는 이 중 **2~5단계(정제·문장 분리·토큰화·형태소 분석)**에 집중합니다. 6단계 벡터화부터는 이후 DAY의 주제입니다(이 글에서는 "왜 숫자로 바꾸는가"만 짚고 넘어갑니다).

### 3.3 전처리가 왜 그렇게 중요한가

강의가 반복해서 강조하는 핵심입니다.

> 🟦 **전처리가 잘못되면, 아무리 좋은 모델을 써도 결과가 잘못된다.** 전처리 = 분석 품질의 기반.

비유하자면 **돈가스를 먹기 좋게 미리 썰어 두는 과정**(p11)입니다. 재료(텍스트)가 아무리 좋아도 썰지 않으면 먹기(분석하기) 어렵습니다.

그렇다고 **무조건 많이 지우는 게 정답은 아닙니다.**

- 마침표(`.`)는 문장의 경계를 나타냅니다 → 무조건 제거하면 문장 분리가 깨집니다.
- `AT&T`의 `&`, 화폐의 `.`/`,`는 의미의 일부입니다.
- 이모티콘도 감정(긍/부정) 정보를 담을 수 있습니다.

즉 전처리는 **"불필요한 것은 지우고, 중요한 것은 지키는"** 균형 작업입니다.

## 4. 텍스트 전처리 5단계 개념 🟦

본격적인 코드에 들어가기 전에, 전처리의 다섯 단계를 한눈에 정리합니다.

| 단계 | 하는 일 | 예시 |
|---|---|---|
| ① 토큰화(Tokenization) | 문장을 작은 단위(토큰)로 쪼갬 | `"나는 학교에 간다"` → `[나는] [학교에] [간다]` |
| ② 정제(Cleaning) | 노이즈(특수문자·HTML·깨진 문자) 제거 | `I'm happy!!! ㅋㅋㅋ` → `I am happy` |
| ③ 정규화(Normalization) | 같은 의미의 표현을 하나로 통일 | `대한민국/한국/코리아` → 하나로, `USA/US` → 하나로, `Python/PYTHON` → `python` |
| ④ 불용어 제거(Stopword) | 의미 전달에 기여 못 하는 단어 제거 | `the, a, of, at` / 한국어 `는, 에` |
| ⑤ 표제어·형태소 | 단어를 기본형/의미 단위로 변환 | `먹었습니다` → `먹다`, `학생들이` → `학생/들/이` |

> 🟩 **정제 vs 정규화 한 줄 구분**
> - **정제**는 *지우는* 작업(노이즈 제거)
> - **정규화**는 *합치는* 작업(다른 표현을 하나로 통일)

이제 이 다섯 단계를 **영어 → 한국어** 순서로 직접 코드로 실행해 봅니다. 영어를 먼저 하는 이유는, 영어가 **띄어쓰기만으로도 단어가 잘 나뉘어** 입문용으로 단순하기 때문입니다.

## 5. 영문 전처리 실습 (NLTK) 🟦

### 5.1 왜 NLTK인가

**NLTK(Natural Language Toolkit)**는 영어 자연어 처리에 가장 널리 쓰이는 파이썬 라이브러리입니다. 토큰화·불용어·품사 태깅 기능을 모두 제공합니다.

```bash
pip install nltk
```

### 5.2 분석 대상 텍스트(코퍼스)

강의는 **스티브 잡스의 2005년 스탠퍼드대 졸업 연설** 일부를 코퍼스(corpus, 분석 대상 텍스트 집합)로 사용합니다.

```python
text = '''Beneath it were the words: "Stay Hungry. Stay Foolish."
It was their farewell message as they signed off.
And I have always wished that for myself.
And now, as you graduate to begin anew, I wish that for you.'''
```

### 5.3 ① 정제 — 특수문자 제거

먼저 파이썬에 어떤 기호들이 "특수문자"로 정의돼 있는지 봅니다.

```python
import string
print(string.punctuation)
```

**실제 출력** (🟩 직접 실행):
```
!"#$%&'()*+,-./:;<=>?@[\]^_`{|}~
```

이제 정규표현식으로 **마침표·공백·일반 문자만 남기고** 나머지 특수문자를 지웁니다.

```python
import re

# [^ ... ] : 대괄호 안 패턴을 '제외'한 나머지를 대상으로
#   \w : 문자(영문자·숫자·밑줄, 그리고 한글도 포함)
#   \. : 마침표
#   \s : 공백
cleaned = re.sub(r'[^\w\.\s]', '', text)   # 마침표/공백/문자 외 제거
cleaned = re.sub(r'\n', ' ', cleaned)      # 줄바꿈 → 공백
print(cleaned)
```

**실제 출력** (🟩 직접 실행):
```
Beneath it were the words Stay Hungry. Stay Foolish. It was their farewell message as they signed off. And I have always wished that for myself. And now as you graduate to begin anew I wish that for you.
```

큰따옴표·쉼표·콜론은 사라지고 **마침표는 남았습니다.** 문장 분리(다음 단계)에서 마침표가 경계 역할을 하기 때문에 일부러 남긴 것입니다.

> 🟩 **`\w`에 대한 오해 주의**: 파이썬 `re`는 기본이 유니코드라 `\w`가 **한글도 포함**합니다. 그래서 똑같은 패턴을 한글 문장에 써도 한글은 보존됩니다(6장에서 활용).

### 5.4 ① 정규화 — 소문자 통일

영어는 `KOREA`, `Korea`, `korea`를 서로 다른 단어로 셉니다. 빈도 계산을 정확히 하려면 소문자로 통일합니다.

```python
tokens = ['Beneath', 'words', 'Stay', 'Hungry', 'Stay']
tokens = [token.lower() for token in tokens]   # 리스트 컴프리헨션
print(tokens)
# ['beneath', 'words', 'stay', 'hungry', 'stay']
```

### 5.5 ② 토큰화 — 문장/단어로 쪼개기

NLTK는 토큰화에 `punkt` 데이터가 필요합니다. 한 번만 내려받으면 됩니다.

```python
import nltk
nltk.download('punkt')        # 구버전
nltk.download('punkt_tab')    # 🟩 최신 NLTK(3.8.2+)는 이 이름이 필요
```

> 🟩 **흔한 에러 예방**: 최신 NLTK에서 `nltk.download('punkt')`만 받고 토큰화하면 `LookupError: punkt_tab not found`가 납니다. 위처럼 **`punkt_tab`도 함께** 받으세요.

```python
from nltk import sent_tokenize, word_tokenize

sent_tokens = sent_tokenize(cleaned)   # 문장 단위
print(sent_tokens)
```

**실제 출력** (🟩 직접 실행):
```
['Beneath it were the words Stay Hungry.',
 'Stay Foolish.',
 'It was their farewell message as they signed off.',
 'And I have always wished that for myself.',
 'And now as you graduate to begin anew I wish that for you.']
```

마침표를 남겨 둔 덕분에 문장이 깔끔하게 5개로 나뉩니다. 다음은 단어 단위입니다.

```python
tokens = word_tokenize(cleaned)        # 단어 단위
print(tokens[:15])
```

**실제 출력** (🟩 직접 실행):
```
['Beneath', 'it', 'were', 'the', 'words', 'Stay', 'Hungry', '.', 'Stay', 'Foolish', '.', 'It', 'was', 'their', 'farewell']
```

> `word_tokenize`는 마침표도 하나의 토큰으로 분리합니다(`'Hungry', '.'`). 단어 빈도 분석 단계에서 이런 기호는 보통 불용어/짧은 단어 제거로 걸러집니다.

### 5.6 ④ 불용어 제거

`the, a, of, at`처럼 자주 나오지만 의미가 약한 단어를 **불용어(stopword)**라고 합니다. NLTK가 영어 불용어 사전을 제공합니다.

```python
nltk.download('stopwords')
from nltk.corpus import stopwords

stop = set(stopwords.words('english'))
print(len(stop), sorted(stop)[:10])
```

**실제 출력** (🟩 직접 실행):
```
198 ['a', 'about', 'above', 'after', 'again', 'against', 'ain', 'all', 'am', 'an']
```

이제 토큰에서 불용어를 빼고, 소문자화한 뒤, **너무 짧은 단어(2글자 이하)**까지 정리합니다.

```python
# ④ 불용어 제거 (대소문자 무시하고 비교)
tokens = [t for t in tokens if t.lower() not in stop]
# 소문자 통일
tokens = [t.lower() for t in tokens]
# 길이 3 이상만 남기기 (마침표·짧은 단어 제거)
tokens = [t for t in tokens if len(t) >= 3]
print(tokens[:12])
```

**실제 출력** (🟩 직접 실행):
```
['beneath', 'words', 'stay', 'hungry', 'stay', 'foolish', 'farewell', 'message', 'signed', 'always', 'wished', 'graduate']
```

`it, were, the, was, their` 같은 불용어와 마침표가 모두 사라지고 **의미 있는 단어만** 남았습니다.

### 5.7 ⑤ 품사 태깅(형태소 분석의 영어판)

영어 형태소 분석은 NLTK의 `pos_tag()`로 단어마다 품사를 붙입니다.

```python
nltk.download('averaged_perceptron_tagger')        # 구버전
nltk.download('averaged_perceptron_tagger_eng')    # 🟩 최신 NLTK 필요
from nltk.tag import pos_tag

tagged = pos_tag(tokens)
print(tagged[:12])
```

**실제 출력** (🟩 직접 실행):
```
[('beneath', 'NN'), ('words', 'NNS'), ('stay', 'VBP'), ('hungry', 'JJ'),
 ('stay', 'NN'), ('foolish', 'JJ'), ('farewell', 'NN'), ('message', 'NN'),
 ('signed', 'VBD'), ('always', 'RB'), ('wished', 'VBN'), ('graduate', 'NN')]
```

태그 읽는 법: `NN`=명사, `NNS`=복수명사, `VB*`=동사, `JJ`=형용사, `RB`=부사.

> 🟩 같은 `stay`가 한 번은 `VBP`(동사), 한 번은 `NN`(명사)으로 태깅됐습니다. **품사는 단어만이 아니라 앞뒤 맥락으로 결정**된다는 걸 보여 줍니다.

> 🟦 **영문 전처리 정리(p22~30)**: 정제(특수문자·소문자) → 토큰화 → 불용어 제거 → 짧은 단어 제거 → 품사 태깅.

## 6. 한글 전처리 실습 (KoNLPy · kss · PyKoSpacing) 🟦

### 6.1 한국어는 왜 더 까다로운가

영어는 띄어쓰기 단위(**어절**)로 잘라도 단어가 대체로 맞습니다. 하지만 한국어는 **조사·어미가 단어에 딱 붙어** 있습니다.

```
"나는 오늘 학교에 갔다"
  띄어쓰기로 자르면 → [나는] [오늘] [학교에] [갔다]     ← '는', '에'가 붙어 있음
  형태소로 자르면   → [나] [는] [오늘] [학교] [에] [가다] ← 의미 단위로 분리
```

그래서 한국어는 **형태소 단위 토큰화**가 기본입니다.

| 구분 | 영어 | 한국어 |
|---|---|---|
| 기본 단위 | 띄어쓰기(어절) | 형태소 |
| 분석 방법 | Word Tokenization | 형태소 분석 |
| 주요 도구 | NLTK, spaCy | KoNLPy(Okt·Kkma·Komoran·Hannanum·Mecab) |

> 🟦 강의의 한·영 비교표(p32)입니다. 형태소 분석 자체는 [7장](#7-형태소-분석)에서 자세히 다루고, 여기서는 그 **앞단계(정제·띄어쓰기·문장 분리·불용어)**를 봅니다.

### 6.2 ① 정제 — 한글 코퍼스에서 특수문자 제거

강의는 같은 잡스 연설의 **한국어 번역문**을 코퍼스로 씁니다.

```python
import re

text = '''그 밑에는 "계속 배고픔을 느끼세요. 계속 바보로 남으세요" 라는 문구가 새겨져 있었습니다.
그들이 전한 마지막 인사말이었습니다. 그리고 저는 항상 제 자신이 그렇길 바랬습니다.'''

cleaned = re.sub(r'[^\w\s]', '', text)   # 문자·공백만 남김(한글은 \w에 포함)
cleaned = re.sub(r'\n', ' ', cleaned)    # 줄바꿈 → 공백
print(cleaned)
```

🟦 **강의 출력 예시(p34)** — 따옴표·마침표가 사라진 한글 문장:
```
그 밑에는 계속 배고픔을 느끼세요 계속 바보로 남으세요 라는 문구가 새겨져 있었습니다 ...
```

> 🟩 5장에서 설명했듯 `\w`가 한글을 포함하므로 **같은 패턴으로 한글이 보존**됩니다.

### 6.3 띄어쓰기 복원 — PyKoSpacing (선택)

한국어 데이터에는 띄어쓰기가 엉망인 경우가 많습니다. 딥러닝 기반 도구 **PyKoSpacing**이 이를 복원합니다.

```python
# PyPI에 없어 git으로 설치
!pip install git+https://github.com/haven-jeon/PyKoSpacing.git

from pykospacing import Spacing
spacing = Spacing()

non_space = re.sub(r'\s', '', text)   # 일부러 띄어쓰기 모두 제거한 텍스트
print(spacing(non_space))
```

🟦 **강의 출력 예시(p35)**:
```
그 밑에는 "계속 배고픔을 느끼세요. 계속 바보로 남으세요"라는 문구가 새겨져 있었습니다.
```

> 🟩 PyKoSpacing은 내부에 딥러닝 모델을 포함해 설치가 다소 무겁습니다. **필수는 아니며**, 띄어쓰기 오류가 많은 데이터에서만 쓰면 됩니다.

### 6.4 ② 토큰화 — 문장 분리(kss)와 어절 분리

한국어 문장 분리는 **kss(Korean Sentence Splitter)**가 표준처럼 쓰입니다.

```python
!pip install kss
import kss

sent_tokens = kss.split_sentences(cleaned)
print(sent_tokens)   # 마침표·문장 구조 기준으로 문장 리스트 반환
```

단순 어절(띄어쓰기) 토큰화는 파이썬 기본 `split()`으로 충분합니다.

```python
def tokenizer(text):
    return text.split()      # 공백 기준 분리 = 어절 토큰화

tokens = tokenizer(cleaned)
print(tokens)   # ['그', '밑에는', '계속', '배고픔을', ...]
```

> 🟩 위 어절 토큰화 결과를 보면 `밑에는`, `배고픔을`처럼 **조사가 붙어** 있습니다. 진짜 의미 단위로 나누려면 [7장의 형태소 분석](#7-형태소-분석)이 필요합니다.

### 6.5 ④ 불용어 제거 — 사용자 사전(CSV)

영어와 달리 한국어는 표준 불용어 사전이 정해져 있지 않아, 보통 **직접 사전(CSV)을 만들어** 씁니다.

```python
import pandas as pd

# stopword_dict.csv 예: 'stopword' 컬럼에 그/라는/바/입니다/하고 ...
stop = pd.read_csv('./data/stopword_dict.csv')
stop_list = list(stop.stopword)
print(stop_list[:5])   # ['그', '라는', '바', '입니다', '하고']

tokens = [t for t in tokens if t not in stop_list]
print(tokens)   # 불용어가 빠지고 의미 있는 단어만 남음
```

> 🟦 강의는 구글 코랩의 `files.upload()`로 CSV를 올립니다(p37). 로컬 PC에서는 그 단계 없이 파일 경로만 맞추면 됩니다.

전체 한글 전처리 흐름을 한 줄로 요약하면:

```
원문 → (정제) → 문장 분리 → 어절/형태소 토큰화 → 불용어 제거 → 분석에 필요한 단어만 남김
```

## 7. 형태소 분석 🟦

### 7.1 형태소란 무엇인가

**형태소(morpheme)**는 **의미를 가지는 가장 작은 언어 단위**입니다. 단어 하나에도 여러 형태소가 들어 있을 수 있습니다.

```
"학생들이"
  학생 → 명사
  들   → (복수) 접미사
  이   → 조사
```

### 7.2 왜 한국어에 꼭 필요한가

한국어는 조사·어미가 매우 다양합니다.

```
먹다 / 먹는다 / 먹었다 / 먹겠습니다  →  모두 동사 "먹다"에서 파생
```

형태소 분석을 하면 이런 변형들을 **같은 의미 계열("먹다")로 묶어** 처리할 수 있습니다. 그래서 검색 엔진, 챗봇, 감성 분석, 문서 분류, LLM 데이터 전처리에 두루 쓰입니다.

### 7.3 품사(Part of Speech)

품사는 단어의 **역할**입니다. 형태소 분석은 보통 "형태소 + 품사"를 함께 돌려줍니다.

| 품사 | 설명 | 예 |
|---|---|---|
| 명사 | 사람·사물 이름 | 오늘, 날씨, 철수 |
| 동사 | 행동 | 달린다, 먹다 |
| 형용사 | 상태 | 좋다 |
| 부사 | 정도 설명 | 매우, 빨리 |
| 조사 | 문법 관계 | 은, 가, 를 |
| 어미 | 문장 종결 | 습니다 |

예) `"철수가 빨리 달린다"` → 철수/명사, 가/조사, 빨리/부사, 달린다/동사.

### 7.4 KoNLPy로 한글 형태소 분석하기

한국어 형태소 분석의 대표 라이브러리는 **KoNLPy**입니다. KoNLPy는 내부적으로 자바(Java)로 만든 분석기들을 감싸고 있어, **자바(JDK)와 JPype1**이 함께 필요합니다.

```bash
pip install JPype1
pip install konlpy
```

> 🟦 KoNLPy가 제공하는 분석기: **Okt, Kkma, Komoran, Hannanum, Mecab**.
> 🟩 그중 **Okt**는 별도 설치 없이 바로 쓸 수 있어 입문용으로 가장 편합니다. **Mecab**은 가장 빠르지만 설치(특히 윈도우)가 까다롭습니다. 아래 예제는 **Okt 기준**으로 작성합니다.

```python
from konlpy.tag import Okt

tagger = Okt()                     # 입문용 기본값
text = "그 밑에는 계속 배고픔을 느끼세요"

# 1) morphs(): 형태소 단위로 분리
print(tagger.morphs(text))

# 2) pos(): 형태소 + 품사 태깅
print(tagger.pos(text))

# 3) nouns(): 명사만 추출
print(tagger.nouns(text))
```

> 🟦 강의는 위 세 함수(`morphs`, `pos`, `nouns`)를 Mecab으로 시연합니다(p45~46). 강의 화면의 `pos()` 출력은 `[('그','MM'), ('밑','NNG'), ('에','JKB'), ('는','JX'), ('계속','MAG'), ('배고픔','NNG'), ('을','JKO'), ('느끼','VV'), ('세요','EP+EF') ...]` 형태였습니다(품사 태그는 분석기마다 다름).
>
> 🟩 **주의 — 출력은 분석기·환경에 따라 달라집니다.** 같은 문장이라도 Okt / Kkma / Mecab의 분리 결과와 품사 표기(`Noun` vs `NNG` 등)가 다릅니다. 위 코드의 출력값은 환경이 준비되면 **직접 실행해 확인**하세요(이 글에서는 임의로 결과를 지어내지 않았습니다).

> 🟩 **분석기 선택 가이드**: 빠른 시작엔 **Okt**, 대용량·속도엔 **Mecab**, 정밀한 품사 분해엔 **Kkma**가 자주 쓰입니다. 강의 자료는 성능 비교에서 *Mecab이 가장 빠르다*고 정리합니다(p50).

### 7.5 (심화) 신조어와 사용자 사전 🟦

형태소 분석기는 **사전에 없는 신조어**에 약합니다. 예를 들어 일부 분석기는 "메타버스"를 "메타 + 버스"로 잘못 쪼갭니다(p48).

해결책(강의 p46·p49):
- **SoyNLP**, **ETRI 형태소 분석 Open API** 등 다른 도구 활용
- **사용자 사전에 신조어 등록**(예: 메타버스, 확장현실(XR), 증강현실(AR))
- **KoalaNLP**: 여러 분석기(KoNLPy, ETRI, Khaiii, Kiwi 등)를 한 인터페이스로 묶어 주는 패키지

> 🟩 ETRI Open API는 **이메일 인증으로 API 키를 발급**받아야 하고, KoalaNLP는 `initialize()` → 분석 → `finalize()` 순서로 자원을 관리합니다. 둘 다 환경 설정이 필요한 **심화 주제**라, 이 글에서는 "신조어는 사용자 사전으로 해결한다"는 개념만 기억하면 충분합니다.

### 7.6 (연결) LLM은 형태소 대신 서브워드를 쓴다 🟦

초기 NLP는 형태소 분석에 크게 의존했지만, 최근 LLM은 **서브워드(subword) 토크나이저**를 많이 씁니다. 단어를 의미가 아니라 **통계적으로 자주 등장하는 조각**으로 나누는 방식입니다.

| 방식 | 아이디어 | 예시 |
|---|---|---|
| **BPE**(Byte Pair Encoding) | 자주 같이 나오는 문자 조합을 한 토큰으로 결합 | `playing → play + ing`, `played → play + ed` |
| **WordPiece** | 자주 쓰이는 단어 조각을 학습 | `unhappy → un + happy` |
| **SentencePiece** | 공백까지 포함해 토큰 생성, 한국어에도 많이 사용 | — |

> 🟩 즉, **형태소 분석은 "언어 규칙 기반", 서브워드는 "데이터 통계 기반"**입니다. 이번 DAY에서 배운 형태소 개념이 다음 단계(벡터화·LLM 토크나이저)를 이해하는 토대가 됩니다.

## 8. 워드 클라우드로 단어 빈도 시각화하기 🟦

숫자 데이터는 축과 단위로 바로 그래프를 그릴 수 있지만, 자연어는 단위가 없어 비교가 어렵습니다. 그래서 **자주 등장한 단어를 크게 표시**하는 **워드 클라우드**가 대표 시각화 방법으로 쓰입니다.

강의가 제시하는 4단계(p51)입니다.

```
1. 데이터 불러오기 → 2. 불필요 문자 제거 → 3. 문장 분석(형태소/명사) → 4. 단어 빈도 계산
```

### 8.1 단어 빈도 계산까지 (실행 가능)

명사만 모았다고 가정하고, 빈도를 세는 부분은 표준 라이브러리로 바로 확인할 수 있습니다.

```python
from collections import Counter

words = ["데이터", "인공지능", "데이터", "분석", "데이터", "모델"]
word_count = Counter(words)
print(word_count)
```

**실제 출력** (🟩 직접 실행):
```
Counter({'데이터': 3, '인공지능': 1, '분석': 1, '모델': 1})
```

한글만 남기는 정제도 정규식 한 줄이면 됩니다(5장의 `\w`와 달리 **한글만** 콕 집습니다).

```python
import re
text = "안녕하세요!!! 오늘 날씨가 정말 좋네요^^ 2026년"
# ㄱ-ㅎ(자음) ㅏ-ㅣ(모음) 가-힣(완성형 한글) 과 공백만 남김
text = re.sub(r"[^ㄱ-ㅎㅏ-ㅣ가-힣\s]", " ", text)
print(text)
```

**실제 출력** (🟩 직접 실행):
```
안녕하세요    오늘 날씨가 정말 좋네요    년
```

`!!!`, `^^`, `2026`이 모두 공백으로 바뀌고 한글만 남았습니다.

### 8.2 워드 클라우드 그리기

```python
!pip install wordcloud
from wordcloud import WordCloud, STOPWORDS
import matplotlib.pyplot as plt

text = open('alice_novel.txt', 'r', encoding='utf-8').read()

wc = WordCloud(max_words=2000, stopwords=set(STOPWORDS)).generate(text)

plt.figure(figsize=(14, 18))   # 너비 14, 높이 18
plt.imshow(wc, interpolation='bilinear')
plt.axis('off')
plt.show()
```

> 🟦 강의 실습은 영어 소설 `alice_novel.txt`와 내장 `STOPWORDS`를 사용합니다(p55~56).
>
> 🟩 **한글 워드 클라우드 주의**: `WordCloud`는 기본 폰트가 영문이라, **한글은 네모(□)로 깨집니다.** 한글을 그리려면 폰트 경로를 지정해야 합니다.

> ```

## 9. 자주 하는 실수

- **특수문자를 전부 지운다** → 마침표(문장 경계), `AT&T`의 `&`, 화폐의 `.`/`,`, 감정 이모티콘처럼 **의미를 가진 기호**까지 날아갑니다. "무엇을 남길지"를 먼저 정하세요.
- **한국어를 띄어쓰기로만 토큰화한다** → `학교에`, `배고픔을`처럼 조사가 붙어 의미 단위가 안 나옵니다. 한국어는 **형태소 분석**이 기본입니다.
- 🟩 **최신 NLTK에서 `punkt`/`pos_tag` 에러** → `punkt_tab`, `averaged_perceptron_tagger_eng`를 추가로 내려받으세요.
- 🟩 **형태소 분석기 출력을 절대값으로 믿는다** → Okt·Kkma·Mecab의 분리 결과와 품사 태그가 서로 다릅니다. 목적에 맞는 분석기를 고르고 결과를 직접 확인하세요.
- 🟩 **한글 워드 클라우드가 네모로 깨진다** → `font_path`로 한글 폰트를 지정해야 합니다.
- 🟩 **불용어를 과하게 지운다** → 분석 목적에 따라 "는/에" 같은 조사도 필요할 수 있습니다. 불용어 사전은 **목적에 맞게** 관리하세요.

## 10. DAY1 핵심 정리

```
자연어 데이터
  - 무엇인가 : 정형 데이터와 달리 구조가 없고, 형태가 다양하며, 맥락에 의존
  - 왜 어려운가 : 중의성 · 오탈자 · 이모티콘 · 문맥 의존성

전처리 5단계
  - 토큰화 → 정제 → 정규화 → 불용어 제거 → 표제어/형태소
  - 정제(지우기) vs 정규화(합치기)를 구분
  - 핵심 원칙 : 불필요한 건 지우고 의미 있는 건 지킨다

언어별 도구
  - 영어  : NLTK (sent/word_tokenize, stopwords, pos_tag)
  - 한국어 : KoNLPy(형태소) + kss(문장) + PyKoSpacing(띄어쓰기)

형태소 분석
  - 의미를 가진 최소 단위 + 품사
  - KoNLPy : morphs() / pos() / nouns(),  입문은 Okt · 속도는 Mecab
  - 신조어 → 사용자 사전 · ETRI · KoalaNLP
  - LLM은 형태소 대신 서브워드(BPE · WordPiece · SentencePiece)

시각화
  - 워드 클라우드 : 빈도가 높은 단어를 크게 표시 (한글은 font_path 필수)
```
