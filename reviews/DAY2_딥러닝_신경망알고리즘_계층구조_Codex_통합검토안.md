## 2. 높은 우선순위의 오류와 수정사항

### 2.1 `nn.Linear`는 퍼셉트론 전체가 아니라 선형 변환 부분이다

**위치:** 초안 2.2~2.4, 특히 “`nn.Linear` 한 줄이 곧 뉴런”

고전적 퍼셉트론은 보통 `가중합 + 편향 + 계단 함수`까지 포함한다. 반면 `nn.Linear`는 공식 문서 기준으로 affine transformation, 즉 `선형 변환 + 편향`만 수행한다. 초안 본문에는 “선형 계산 부분”이라는 설명이 있지만 제목과 일부 문장은 `nn.Linear` 자체가 완전한 퍼셉트론인 것처럼 읽힌다.

다음처럼 구분해야 한다.

```text
nn.Linear : z = Wx + b를 계산하는 선형 계층
퍼셉트론  : z를 계산한 뒤 계단 함수로 0/1을 결정하는 모델
일반 신경망 뉴런: 선형 계산 뒤 ReLU, Sigmoid 같은 활성화 함수를 적용
```

공식 근거: [PyTorch 2.12 `nn.Linear`](https://docs.pytorch.org/docs/2.12/generated/torch.nn.Linear.html)

### 2.2 단일 샘플 수식과 PyTorch 배치 shape를 구분해야 한다

**위치:** PDF 20쪽, 초안 5장과 7장의 shape 설명

PDF 20쪽 그림은 입력 행렬 `X: (m, n)`, 가중치 `W: (n, 1)`, 편향과 출력 `(m, 1)`을 보여 주지만 초안은 이를 `z = Wx + b` 한 줄로만 정리했다. 수학 교재마다 열벡터와 행벡터 관례가 다르므로 입문자는 `Wx`와 `XW` 중 어느 쪽이 맞는지 혼란스러울 수 있다.

PyTorch `nn.Linear(in_features, out_features)`의 저장 가중치는 `(out_features, in_features)`이고, 배치 입력의 마지막 차원에 다음 계산을 적용한다.

```text
입력 X       : (batch_size, in_features)
weight       : (out_features, in_features)
bias         : (out_features,)
출력 Z       : (batch_size, out_features)
개념적 계산  : Z = X @ weight.T + bias
```

`nn.Linear(10, 64)`에서 `(8, 10) → (8, 64)`가 되는 이유를 이 식과 연결하면 PDF 20쪽의 행렬 그림도 제대로 회수된다.

### 2.3 logit, 확률, 최종 라벨을 분리해야 한다

**위치:** 초안 5.1 출력층 표, 6.2 손실 함수, 8.3 실무 추천

“이진 분류 출력층은 0 또는 1”, “문장 생성은 다음 단어 확률”이라는 설명은 최종 해석만 말한 것이다. 실제 학습 중 모델은 보통 가공 전 점수인 **logit**을 출력한다.

| 문제 | 모델의 학습 출력 | 라벨 형식 | 손실 | 사람이 해석할 때 |
|---|---|---|---|---|
| 회귀 | `(N, 1)` 실수 | `float32`, `(N, 1)` | `MSELoss` 등 | 그대로 예측값 |
| 이진 분류 | `(N, 1)` logit | 0/1 `float32`, 같은 shape | `BCEWithLogitsLoss` | `sigmoid` 후 임계값 적용 |
| 다중 분류 | `(N, C)` logits | 클래스 인덱스 `torch.long`, `(N,)` | `CrossEntropyLoss` | `softmax` 후 확률, `argmax` 후 클래스 |

현재 AND 코드는 `(4, 1)` float 라벨과 `BCEWithLogitsLoss`를 사용하므로 올바르다. 다만 손실 함수 표에서 `BCELoss`를 함께 소개하려면 다음 짝도 명시해야 한다.

```text
Sigmoid 출력 + BCELoss
raw logits + BCEWithLogitsLoss
```

입문 글에서는 수치 안정성이 좋은 두 번째 조합을 기본으로 두는 편이 낫다.

공식 근거:

- [PyTorch 2.12 `BCEWithLogitsLoss`](https://docs.pytorch.org/docs/2.12/generated/torch.nn.BCEWithLogitsLoss.html)
- [PyTorch 2.12 `CrossEntropyLoss`](https://docs.pytorch.org/docs/2.12/generated/torch.nn.CrossEntropyLoss.html)

### 2.4 일반 학습 골격에 validation/test와 평가 모드가 빠져 있다

**위치:** 초안 6.5 “거의 모든 PyTorch 딥러닝 학습 코드는 이 구조”

현재 코드는 `X_train`, `y_train`의 훈련 손실만 출력한다. PDF의 기본 구조를 충실히 옮긴 것이지만, 이것을 “거의 모든 학습 코드”라고 소개하면 모델 선택과 최종 평가도 훈련 데이터로 하는 것으로 오해할 수 있다.

다음 원칙을 짧게 추가해야 한다.

1. **train**: 가중치를 학습한다.
2. **validation**: epoch, 학습률, 모델 구조를 선택하고 과적합을 확인한다.
3. **test**: 모든 선택이 끝난 뒤 한 번만 최종 성능을 확인한다.
4. 학습 때 `model.train()`, 검증·예측 때 `model.eval()`을 사용한다.
5. 검증·예측은 `torch.no_grad()` 또는 `torch.inference_mode()` 안에서 수행한다.

현재 AND 진리표는 가능한 입력 네 개 전체를 학습하는 **논리회로 재현 예제**이므로 억지로 train/test로 나눌 필요가 없다. 대신 “일반화 성능을 추정하는 예제가 아니라 네 행의 진리표를 학습할 수 있는지 확인하는 예제”라고 밝혀야 한다.

현재 예제에는 scaler, imputer, encoder 같은 학습형 전처리가 없어 데이터 누수는 발생하지 않는다. 이후 실제 데이터 예제로 확장할 때는 데이터를 먼저 분리하고 전처리 통계를 train에만 맞춰야 한다.

공식 근거:

- [PyTorch 2.12 `Module.eval()`](https://docs.pytorch.org/docs/2.12/generated/torch.nn.Module.html#torch.nn.Module.eval)
- [PyTorch 2.12 `torch.no_grad`](https://docs.pytorch.org/docs/2.12/generated/torch.no_grad.html)

### 2.5 활성화 함수 비교표의 절대 표현과 출력 범위를 정정해야 한다

**위치:** 초안 8.2~8.3, PDF 31~40쪽

이 부분은 PDF 자체에 과장과 오류가 많고 초안이 일부를 그대로 유지했다.

- Step은 단순히 “미분 불가능”이라기보다 `x=0`에서 미분 불가능하고 나머지 구간의 기울기가 0이므로 경사 기반 학습에 적합하지 않다.
- Tanh는 “Sigmoid의 개선 버전”이나 “항상 더 빠름”으로 단정할 수 없다. 출력이 0 중심이라는 차이가 있을 뿐이다.
- ReLU는 기울기 소실을 완전히 없애지 않는다. 양수 구간의 기울기가 1이어서 sigmoid/tanh의 포화 문제를 줄이지만, 음수 구간은 기울기가 0이고 깊은 네트워크 전체에서는 여전히 기울기 소실·폭주가 생길 수 있다.
- Leaky ReLU는 Dead ReLU 위험을 **완화**하지만 해결을 보장하지 않는다.
- SELU의 자기 정규화 효과는 초기화, 네트워크 구조, `AlphaDropout` 등 조건이 맞을 때 기대할 수 있다.
- GELU, SiLU, Mish의 출력 범위를 `-∞~∞`로 쓰면 안 된다. 이 함수들은 음수 쪽에 유한한 최솟값이 있다.
- “GELU는 Transformer/LLM 표준”은 너무 넓은 주장이다. BERT 계열과 일부 Transformer에서 널리 쓰이지만, 여러 현대 LLM은 SiLU 계열이나 gated activation을 사용한다.
- “Swish가 ReLU보다 성능 우수”도 모든 데이터와 모델에서 성립하지 않는다.
- “속도: 매우 빠름/느림”, “가장 많이 사용”은 하드웨어와 구현, 시점에 따라 달라지는 정보다.

출력 범위와 사용처를 정확히 유지하기 어렵다면 비교표에서 `기울기 소실`, `속도`, `현재 사용처`의 단정적 등급을 줄이고 “핵심 성질” 중심으로 바꾸는 편이 안전하다.

### 2.6 Gradient와 가중치를 ‘책임’ 또는 ‘중요도’로 바로 해석하면 안 된다

**위치:** 초안 2.2, 6.3

가중치는 입력 단위와 전처리, 다른 가중치, 활성화 함수에 따라 의미가 바뀐다. 초안의 주의 문구는 좋지만 바로 앞 정의인 “값이 클수록 그 입력이 결과에 더 큰 영향을 준다”도 여전히 절대적으로 읽힌다.

Gradient는 현재 미니배치와 현재 파라미터 지점에서 손실이 얼마나 민감하게 변하는지를 나타낸다.

- 절댓값: 현재 지점의 국소적 민감도
- 부호: 파라미터를 어느 방향으로 바꾸면 손실이 증가하는지
- 업데이트: `w ← w - learning_rate × gradient`

Gradient가 크다고 해서 그 특성이 전역적으로 가장 중요하거나 인과적으로 가장 책임이 있다는 뜻은 아니다.

## 4. 더 자세히 설명할 내용

### 4.1 출력층의 값은 곧바로 0/1이 아니다

AND 예제에서 `model(X)`는 확률이 아닌 logit이다. `torch.sigmoid()`를 적용해야 0~1 확률로 바뀌며, 다시 `>= 0.5` 같은 기준을 적용해야 최종 0/1 라벨이 된다. 이 세 단계를 예측 코드로 보여 주면 손실 함수 설명과 자연스럽게 연결된다.

### 4.2 dtype과 라벨 shape

현재 AND 예제는 소수점 리터럴을 사용해 `X`와 `y`가 `float32`가 되므로 정상이다. 본문에 다음 한 줄을 추가하면 좋다.

```text
BCEWithLogitsLoss: 출력과 라벨은 같은 shape의 부동소수점 텐서
CrossEntropyLoss: 출력은 (N, C) logits, 라벨은 (N,) torch.long 클래스 인덱스
```

### 4.3 학습률

초안은 `lr=0.01`, `lr=0.001`을 사용하지만 `lr`가 무엇인지 설명하지 않는다. 너무 크면 손실이 튀거나 발산할 수 있고, 너무 작으면 학습이 느리거나 충분히 진행되지 않을 수 있다는 정도를 추가하면 된다.

### 4.4 평가 지표

현재 AND 예제에서 손실 감소만 보고 “학습 성공”이라고 판단한다. 이 예제는 네 입력의 예측 라벨이 진리표와 모두 일치하는지도 확인하는 편이 명확하다.

실제 문제로 확장할 때는 과제에 맞는 지표가 필요하다.

| 문제 유형 | 입문 단계에서 함께 볼 지표 |
|---|---|
| 이진·다중 분류 | accuracy, 클래스 불균형 시 precision/recall/F1 |
| 회귀 | MAE 또는 RMSE, 필요하면 R² |
| 딥러닝 공통 | train loss와 validation loss 곡선 |

이 글에는 실제 데이터셋의 성능을 주장하는 모델이 없으므로 복잡한 지표 실습까지 넣을 필요는 없다. 원칙과 AND 정확도 확인만 추가해도 충분하다.

### 4.5 은닉층의 특징 학습 설명

“첫 층은 선, 다음 층은 눈·코·입, 깊은 층은 얼굴”은 CNN 시각화에서 자주 쓰는 직관이지만 모든 신경망과 모든 뉴런이 정확히 이런 의미를 학습한다는 보장은 없다.

다음처럼 완화하는 편이 좋다.

> 이미지 모델에서는 얕은 층이 비교적 단순한 패턴에, 깊은 층이 더 복합적인 패턴에 반응하는 경향이 관찰됩니다. 다만 각 층의 표현은 모델 구조와 데이터에 따라 달라집니다.

### 4.6 CPU, dtype, device

초안은 CUDA를 직접 사용하지 않으므로 현재 코드는 CPU에서 일관되게 실행된다. device 코드를 억지로 추가할 필요는 없다. 이후 GPU 예제를 넣는다면 모델과 입력·라벨을 모두 같은 device로 이동해야 한다는 원칙만 연결하면 된다.

## 7. 바로 붙여 넣을 수 있는 수정 블록

### 7.1 `nn.Linear`와 배치 shape 교체안

````markdown
### PyTorch의 `nn.Linear`는 무엇을 계산할까?

`nn.Linear(in_features, out_features)`는 퍼셉트론 전체가 아니라
가중합과 편향인 **선형 변환 부분**을 계산합니다.

$$
z = Wx + b
$$

고전적인 퍼셉트론은 이 값에 계단 함수를 적용해 0 또는 1을 결정하고,
일반 신경망은 ReLU나 Sigmoid 같은 활성화 함수를 이어 붙입니다.

PyTorch에서 배치 입력을 사용할 때 shape는 다음과 같습니다.

```text
입력 X       : (batch_size, in_features)
weight       : (out_features, in_features)
bias         : (out_features,)
출력 Z       : (batch_size, out_features)
개념적 계산  : Z = X @ weight.T + bias
```

예를 들어 `nn.Linear(10, 64)`는 `(8, 10)` 입력을 `(8, 64)`로 바꿉니다.
배치 크기 8은 유지되고 마지막 특징 차원만 10에서 64로 바뀝니다.
````

### 7.2 출력층·라벨·손실 함수 교체안

```markdown
### 출력층과 손실 함수는 함께 정한다

모델의 학습 출력, 라벨 형식, 손실 함수는 서로 맞아야 합니다.

| 문제 | 모델 출력 | 라벨 | PyTorch 손실 |
|---|---|---|---|
| 회귀 | 실수값 `(N, 1)` | `float32`, `(N, 1)` | `nn.MSELoss()` 등 |
| 이진 분류 | raw logit `(N, 1)` | 0/1 `float32`, 같은 shape | `nn.BCEWithLogitsLoss()` |
| 다중 분류 | logits `(N, C)` | 클래스 번호 `torch.long`, `(N,)` | `nn.CrossEntropyLoss()` |

`BCEWithLogitsLoss`는 Sigmoid와 이진 교차엔트로피를 함께 계산하므로
모델 끝에 Sigmoid를 붙이지 않습니다. 예측을 사람이 확률로 볼 때만
`torch.sigmoid(logits)`를 적용합니다.

`CrossEntropyLoss`도 raw logits를 받습니다. 학습 전에 Softmax를 적용하지 않고,
추론 때 확률이 필요할 때만 `torch.softmax(logits, dim=1)`을 사용합니다.
```

### 7.3 AND 예측과 평가 추가안

```python
model.eval()
with torch.no_grad():
    logits = model(X)
    probabilities = torch.sigmoid(logits)
    predicted = (probabilities >= 0.5).float()
    accuracy = (predicted == y).float().mean()

print(probabilities.flatten())
print(predicted.flatten())
print(f"진리표 정확도: {accuracy.item():.2%}")
```

```text
이 예제의 네 행은 가능한 AND 입력 전체입니다. 따라서 여기서 정확도 100%는
네 행의 진리표를 재현했다는 뜻이지, 새로운 현실 데이터에 대한 일반화 성능을
검증했다는 뜻은 아닙니다.
```

### 7.4 Gradient와 학습률 교체안

```markdown
Gradient는 현재 파라미터 값에서 손실이 얼마나 민감하게 변하는지를 나타냅니다.
절댓값이 크면 작은 파라미터 변화에도 현재 손실이 크게 변할 수 있고,
부호는 손실이 증가하는 방향을 알려 줍니다.

가장 기본적인 업데이트는 다음과 같습니다.

$$
w \leftarrow w - \eta \frac{\partial L}{\partial w}
$$

- $L$: 손실
- $\frac{\partial L}{\partial w}$: 가중치에 대한 gradient
- $\eta$: learning rate(학습률)

학습률이 너무 크면 최솟값을 지나치거나 발산할 수 있고,
너무 작으면 학습이 매우 느려질 수 있습니다.

Gradient가 크다고 해서 그 입력이 전역적으로 가장 중요하거나
오차에 인과적으로 가장 책임이 있다는 뜻은 아닙니다.
```

### 7.5 train/validation/test 추가안

```markdown
> 📌 **보충 — 실제 데이터에서는 세 데이터의 역할을 나눕니다**
>
> - **train**: 모델의 가중치를 학습합니다.
> - **validation**: 모델 구조, 학습률, epoch 등을 선택하고 과적합을 확인합니다.
> - **test**: 모든 선택이 끝난 뒤 최종 성능을 한 번 확인합니다.
>
> 학습할 때는 `model.train()`, 검증과 예측 때는 `model.eval()`을 사용합니다.
> 검증·예측 계산은 `torch.no_grad()` 안에서 수행해 불필요한 gradient 저장을 막습니다.
> 표준화나 결측치 대체처럼 데이터에서 규칙을 배우는 전처리는 train에만 `fit`하고,
> validation과 test에는 같은 규칙을 적용만 해야 합니다.
```

### 7.6 활성화 함수 설명 교체안

```markdown
| 함수 | 핵심 성질 | 주의점과 대표 용도 |
|---|---|---|
| Step | 0 또는 1 출력 | 거의 모든 구간의 기울기가 0이라 역전파 학습에 부적합 |
| Sigmoid | 0~1 출력 | 이진 확률 해석에 사용, 큰 절댓값에서 포화 |
| Tanh | -1~1 출력, 0 중심 | 역시 큰 절댓값에서 포화 |
| ReLU | 음수 0, 양수 그대로 | 계산이 단순하지만 음수 구간 기울기는 0 |
| Leaky ReLU | 음수에도 작은 기울기 | Dead ReLU 위험을 완화하지만 해결을 보장하지 않음 |
| GELU | 입력을 부드럽게 조절 | BERT 계열 등 일부 Transformer에서 사용 |
| SiLU | `x * sigmoid(x)` | 여러 현대 모델에서 사용되지만 항상 ReLU보다 좋은 것은 아님 |

어떤 활성화 함수가 좋은지는 모델 구조와 데이터에 따라 달라집니다.
입문 단계에서는 은닉층의 출발점으로 ReLU를 사용하고,
출력층은 문제 유형과 손실 함수에 맞춰 선택하면 충분합니다.
```
