## 2. 높은 우선순위의 오류와 수정사항

### High 1. GNMT 개념도에서 `je suis étudiant`와 `moi suis étudiant`가 섞임

위치: 초안 §5.2, 특히 154행, 166행 주변

문제:

- PDF p.17-p.20의 다이어그램은 인코더 입력 `I am a student`, 디코더 입력/정답이 `moi suis étudiant </s>`로 표시됩니다.
- PDF p.18 본문은 별도 설명 예시로 `Je t'aime`을 사용합니다.
- PDF p.19 본문은 첫 번째 출력 위치의 후보 확률 예시로 `Je: 0.80, Tu: 0.10, Il: 0.05, 기타: 0.05`를 듭니다.
- 초안은 "디코더가 `je suis étudiant`를 생성"한다고 쓰면서, 바로 아래 코드 블록은 `moi suis étudiant`로 적었습니다. 이 상태는 입문자가 "다이어그램의 정답이 je인지 moi인지" 혼동할 수 있습니다.

수정 방향:

- 다이어그램 설명은 `moi suis étudiant </s>`로 고정합니다.
- `Je t'aime`은 PDF 본문 설명 예시, `Je/Tu/Il/기타`는 확률 후보 예시라고 분리합니다.
- 사용자 요청에서 언급된 `je suis étudiant`는 PDF 이미지 기준으로는 선택된 다이어그램 출력이 아닙니다. 다이어그램의 선택 단어는 첫 위치 `moi(0.4)`, 두 번째 `suis(0.6)`, 세 번째 `étudiant(0.5)`, 네 번째 `</s>(0.6)`입니다.

### 7.2 §7.3 Keras 인코더/디코더 LSTM 예시

API 사용 판단:

| 항목 | 초안 코드 | 검토 |
|---|---|---|
| 인코더 `LSTM(return_state=True)` 반환값 | `_, state_h, state_c = encoder_lstm(encoder_inputs)` | 맞음. `return_sequences=False` 기본값에서는 마지막 출력, `state_h`, `state_c` 순서 |
| 디코더 `initial_state` | `initial_state=encoder_states` | 맞음. LSTM에는 `[state_h, state_c]` 전달 |
| 디코더 `return_sequences=True` | 사용함 | 맞음. 매 시점 단어 예측에 필요 |
| 디코더 `return_state=True` | 사용함 | 학습 모델에서는 상태를 버려도 되지만 추론 모델을 만들 때 필요하므로 허용 가능 |
| Dense 출력층 | `Dense(tar_vocab_size, activation="softmax")` | 맞음. 마지막 축에 vocab 확률 분포 생성 |
| 손실 함수 | `categorical_crossentropy` | 원-핫 타깃일 때 맞음 |
| sparse 손실 설명 | 표로 설명 | 맞음. 정수 인덱스 타깃이면 `sparse_categorical_crossentropy` |

미실행 표시 판단:

- 현재 환경에 TensorFlow/Keras가 설치되어 있지 않으므로 미실행으로 표시한 처리는 적절합니다.
- 초안은 출력값을 지어내지 않았고, "참고용 코드"라고 표시했습니다. 이는 유지해야 합니다.
- 단, 최종본에서는 코드가 "완전 실행 예제"가 아니라 구조 설명용 partial snippet임을 더 분명히 하면 좋습니다.

### 8.2 §5.2 확률표 설명 수정 블록

````markdown
🟦 디코더는 각 위치에서 가능한 여러 단어 후보의 확률을 계산합니다. PDF 다이어그램에서는 첫 번째 위치에서 `moi`가 0.4로 선택되고, 이어서 `suis`, `étudiant`, `</s>`가 선택되는 흐름을 보여 줍니다. 본문에는 별도 예시로 다음과 같은 후보 확률도 제시됩니다.

```text
[1번째 출력 위치] Je : 0.80, Tu : 0.10, Il : 0.05, 기타 : 0.05
→ 가장 높은 Je 선택
```

> 🟩 보충: 이런 후보 확률 분포를 만들 때 보통 `softmax`를 쓰고, 가장 큰 값을 고르는 동작을 `argmax`라고 부릅니다. 실제 번역 시스템은 매 위치에서 가장 큰 단어 하나만 고르지 않고 여러 후보 문장을 함께 탐색하는 방식도 사용합니다.
````
