## 높은 우선순위의 오류와 수정사항

### High 1. PDF에 없는 보충 용어가 일부 🟦 원문처럼 보임

위치:

- 초안 `6.1 디코더 입력 = 정답을 한 칸씩 밀어 넣기 (Teacher Forcing) 🟦`
- 초안 `6.2 디코더 출력 = 단어별 확률 (softmax) 🟦`
- 초안 요약의 `GNMT(2016) -> Attention -> Transformer -> 현재 LLM 번역`
- 초안 요약의 `softmax(합=1) -> argmax`

문제:

PDF는 "정답 문장을 한 칸씩 밀어서 디코더 입력으로 사용"하는 흐름을 설명하지만 `Teacher Forcing`이라는 용어를 직접 쓰지 않는다. PDF는 각 출력값의 합이 1인 확률을 설명하지만 `softmax`나 `argmax`라는 용어를 직접 쓰지 않는다. 2016년 GNMT 도입과 Transformer 연결도 초안에서 추가한 보충이다.

왜 중요한가:

이 글은 🟦를 "강의 PDF에 직접 나온 개념·예시·비교"로 정의했다. 제목에 보충 용어를 넣고 🟦를 붙이면 독자가 해당 용어까지 PDF 원문이라고 오해할 수 있다.

수정 방향:

- PDF 원문 흐름은 🟦로 유지한다.
- `Teacher Forcing`, `softmax`, `argmax`, `2016년`, `Transformer 연결`은 🟩 보충으로 분리한다.
- 제목에서는 보충 용어를 빼거나 제목 자체를 🟦+🟩 혼합으로 명확히 표시한다.

### High 2. GNMT와 Attention의 역사적 관계를 약간 더 정확히 표현해야 함

위치:

- 초안 §4.2 보충: "Encoder-Decoder(Seq2Seq) 아이디어는 이후 Attention -> Transformer로 이어졌고..."
- 초안 §8 요약: "GNMT(2016) -> Attention -> Transformer -> 현재 LLM 번역"

문제:

입문용 흐름으로는 큰 문제 없지만, 공식 GNMT 논문 기준으로 2016년 GNMT는 단순 LSTM 인코더-디코더만이 아니라 attention과 residual connection을 포함한 깊은 LSTM encoder-decoder 모델이다. 따라서 "GNMT 다음에 Attention이 나왔다"처럼 읽히면 역사적으로 부정확하다.

수정 방향:

- "강의에서는 GNMT를 RNN 기반 Seq2Seq로 단순화해 설명한다"라고 두고,
- 보충에서는 "실제 Google GNMT(2016)는 LSTM 기반 encoder-decoder에 attention을 포함한 시스템이었다. 이후 Transformer 계열 모델에서는 attention이 RNN을 대체하는 핵심 구조로 더 중심이 되었다"라고 정리하는 편이 정확하다.

### 3. `argmax`는 강의 수준 설명으로 맞지만 실제 번역 시스템은 더 복잡할 수 있음

초안은 챗봇이 이전 입력과 현재 입력을 함께 고려할 수 있다고 설명한다. PDF p15-p16과 일치한다. 다만 RNN 기반 모델은 긴 대화 맥락을 완벽히 기억한다고 단정하지 않도록, "짧은 문맥을 반영할 수 있지만 긴 문맥에서는 한계가 있다"는 보충을 넣으면 좋다.

### 블록 1. §6.1 제목과 Teacher Forcing 출처 분리

```markdown
### 6.1 디코더 입력 = 정답을 한 칸씩 밀어 넣기 🟦

🟦 영어 → 불어 예: 원문 `I love you`, 정답 `Je t'aime`.

- **인코더 입력**: 번역할 원문(`I love you`)을 순서대로 읽어 의미 벡터로 압축합니다.
- **디코더 입력**: 학습 단계에서 **정답 문장을 한 칸씩 밀어서** 넣습니다. 시작 토큰 뒤에 정답 단어를 차례로 주고, **다음 단어를 맞히도록** 학습합니다.
- **디코더 출력**: 한 번에 전체 문장이 아니라 **한 단어씩 순서대로** 생성합니다. (`Je → t' → aime`)

> 🟩 **용어 보충 — Teacher Forcing**: 위처럼 학습할 때 디코더에 **직전의 정답 단어**를 넣어 다음 단어를 맞히게 하는 방식을 Teacher Forcing이라고 부릅니다. 실제 번역처럼 정답이 없는 추론 단계에서는 디코더가 **직전에 스스로 생성한 단어**를 다음 입력으로 사용합니다.
```

### 블록 2. §6.2 제목과 softmax/argmax 출처 분리

````markdown
### 6.2 디코더 출력 = 단어별 확률 🟦

🟦 디코더는 각 위치에서 **가능한 여러 단어 후보의 확률**을 계산하고, 그 **확률의 합은 1**이 됩니다. 강의에서는 그중 **가장 확률이 높은 단어**를 선택해 번역 결과를 만든다고 설명합니다.

🟦 예(첫 번째 출력 위치):

```text
Je : 0.80,  Tu : 0.10,  Il : 0.05,  기타 : 0.05   (합 = 1)
→ 가장 높은 Je 선택
```

> 🟩 **용어 보충**: 딥러닝에서는 이런 확률 분포를 만들 때 보통 `softmax`를 사용하고, 가장 큰 값을 고르는 동작을 `argmax`라고 부릅니다. 실제 번역 시스템은 가장 큰 단어 하나만 고르지 않고 여러 후보 문장을 함께 탐색하는 방식도 사용할 수 있습니다.
````
