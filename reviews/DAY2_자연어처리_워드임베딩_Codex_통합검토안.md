## 4. 더 자세히 설명할 내용

### 4.1 TF-IDF와 Word2Vec의 역할 차이

초안은 비교표가 있어 좋지만, 본문 중간의 "TF-IDF로 본 분포 가설의 가장 단순한 형태"는 입문자에게 TF-IDF와 Word2Vec의 경계를 흐릴 수 있습니다. 이미 "Word2Vec이 아님"이라고 주의 문구를 넣은 점은 좋습니다. 여기에 한 문장을 더해 "문서 단위 동시 등장 기반 유사도"라고 못 박으면 안전합니다.

### 4.2 LDA 결과 해석의 한계

4개 문서로 LDA를 돌리는 예제는 학습용으로는 좋지만, 실제 토픽 모델링이라고 보기에는 데이터가 지나치게 작습니다. "라벨 없이도 스스로 찾았다"는 표현은 유지하되, 바로 뒤에 "이 예제는 사람이 이해하기 쉬운 장난감 데이터라 깔끔하게 나뉜다"는 단서를 넣으세요.

### 4.3 train/validation/test와 데이터 누수

이번 PDF와 초안의 실습은 탐색적 벡터화/토픽 모델링이라 train/test split이 필수는 아닙니다. 그러나 이 벡터가 분류 모델의 입력으로 들어간다면 `TfidfVectorizer`, `CountVectorizer`, LDA, PCA 같은 학습형 전처리는 반드시 훈련 데이터에만 `fit`해야 합니다. 이 주의는 입문자에게 매우 유용합니다.

### 4.4 평가 지표

초안은 예측 모델이 아니라 표현/탐색 예제이므로 accuracy, F1 같은 분류 지표가 없어도 됩니다. 다만 LDA는 주제 번호를 정답처럼 비교하면 안 되며, 상위 단어 해석, 문서별 주제 분포, 필요하면 coherence/perplexity 같은 보조 지표를 쓸 수 있다고 한 줄 덧붙이면 좋습니다.

## 5. 유용한 추가 내용

- 한국어 TF-IDF에서 `token_pattern=r"(?u)\b\w+\b"`를 지정한 설명은 좋습니다. scikit-learn 공식 문서의 기본값이 2글자 이상 토큰을 선택한다는 점과 맞습니다.
- `random_state=42`를 LDA에 지정한 점은 재현성 측면에서 좋습니다.
- Word2Vec 출력값을 지어내지 않고 `gensim` 미설치와 작은 코퍼스 한계를 밝힌 점은 매우 좋습니다.
- t-SNE의 `perplexity`가 표본 수보다 작아야 한다는 주의는 공식 문서와도 맞습니다.
- PDF의 역사 설명 중 TF-IDF와 GloVe 연도 보충을 `🟩`로 구분한 점은 적절합니다. PDF 원문처럼 말하지 않고 별도 보충으로 처리했기 때문입니다.

## 7. 바로 붙여 넣을 수 있는 수정 블록

### 7.1 3.1 코사인 유사도 실습 대체 블록

아래 블록은 `X`를 먼저 만들기 때문에 3.1에 그대로 넣어도 실행됩니다.

````markdown
`scikit-learn`으로 문서 간 코사인 유사도를 직접 계산해 봅니다. 아래 코드는 이해를 위한 작은 리뷰 4개를 먼저 TF-IDF 벡터로 바꾼 뒤, 그 벡터끼리의 각도를 비교합니다.

```python
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity

corpus = [
    "배송 빠르다 포장 꼼꼼 만족",
    "배송 느리다 포장 엉망 실망",
    "가격 저렴 가격 합리 품질 만족",
    "가격 비싸다 품질 별로 실망",
]

vec = TfidfVectorizer(token_pattern=r"(?u)\b\w+\b")
X = vec.fit_transform(corpus)

sim = cosine_similarity(X)
print(sim.round(3))
```

해석: doc0 기준으로는 doc1이 가장 가깝습니다(둘 다 배송/포장 이야기). 전체 쌍 중에서는 doc2-doc3이 0.396으로 가장 높게 나옵니다(둘 다 가격/품질 이야기). 이 예제는 작은 장난감 데이터이므로, 실제 서비스에서는 더 큰 코퍼스에서 임계값을 다시 정해야 합니다.
````

### 7.2 TF-IDF 공식 설명 보강 블록

```markdown
> 🟩 **구현 주의**: 위 공식은 강의에서 직관을 설명하기 위한 기본형입니다. `scikit-learn`의 `TfidfVectorizer`는 기본적으로 `smooth_idf=True`를 사용해 `idf = log((1 + N) / (1 + df)) + 1`처럼 보정합니다. 그래서 모든 문서에 등장한 단어도 구현상 가중치가 정확히 0이 되지는 않습니다. 핵심은 "공통적으로 너무 자주 나오는 단어의 영향력을 낮춘다"는 방향으로 이해하면 됩니다.
```

### 7.3 LDA 작은 데이터 주의 블록

```markdown
> 🟩 **작은 데이터 주의**: 이 예제는 원리를 보여 주기 위한 4문장짜리 장난감 코퍼스라 주제가 깔끔하게 나뉩니다. 실제 LDA는 훨씬 많은 문서에서 상위 단어, 문서별 주제 분포, 사람이 붙인 주제 이름을 함께 보며 해석해야 합니다. 주제 번호 0, 1 자체에는 순서나 우열이 없습니다.
```

### 7.4 데이터 누수 주의 블록

```markdown
> 🟩 **데이터 누수 주의**: 이번 예제처럼 탐색적으로 문서 구조를 보는 경우에는 train/test split이 필수는 아닙니다. 하지만 TF-IDF, LDA, PCA 결과를 나중에 분류 모델의 입력으로 쓸 때는 `fit`을 훈련 데이터에만 해야 합니다. 검증/테스트 데이터에는 이미 학습한 벡터라이저나 모델로 `transform`만 적용해야 평가가 공정합니다.
```

| 중간 | Word2Vec 유추 주석이 현재 코퍼스와 불일치 | 6.2 | KeyError 가능성과 사전 학습 모델 필요성 명시 |
| 낮음 | p9의 AI 자동화 방향 문장 누락 | 5.4 | 선택적으로 한 문장 추가 |
| 낮음 | 최종 게시용 메타 주석 제거 | 1~66행 | 최종본에서 삭제 |

## 9. 딥러닝/프레임워크/평가 점검

- 프레임워크 혼용: 초안은 scikit-learn 중심이고, Word2Vec에서만 gensim으로 전환합니다. 전환 사실을 명시했으므로 큰 문제는 없습니다.
- Tensor shape/dtype/device: PyTorch/Keras 텐서 학습 코드가 없으므로 대부분 비해당입니다. 다만 `model.wv["가격"].shape`가 `(100,)`임을 보여 주는 것은 좋습니다.
- 출력층/라벨/손실함수: 분류용 딥러닝 모델이 없으므로 비해당입니다.
- 데이터 누수: 현재는 탐색/비지도 예제라 split 필수는 아니지만, 분류 모델로 확장할 경우 vectorizer/LDA/PCA는 훈련 데이터에만 `fit`해야 합니다.
- 평가 지표: 지도학습 모델이 아니므로 accuracy/F1은 필요 없습니다. LDA는 주제 상위 단어와 문서-주제 분포 해석이 중심이고, 필요 시 coherence/perplexity는 선택 보충입니다.
- 설치/API: `TfidfVectorizer`, `CountVectorizer`, `TSNE`, `Word2Vec`은 버전에 따라 기본값/매개변수명이 바뀔 수 있으므로 게시 전 공식 문서 링크 확인을 유지하세요.

## 10. 공식 문서 확인 메모

- scikit-learn `TfidfVectorizer`: 공식 문서의 기본 `token_pattern`은 `(?u)\b\w\w+\b`로 2글자 이상 토큰을 고릅니다. 초안의 한국어 1글자 토큰 보존 설명은 타당합니다.
- scikit-learn `TSNE`: 공식 문서에서 `perplexity`는 표본 수보다 작아야 하며, 보통 5~50 범위 선택을 언급합니다. 초안의 t-SNE 주의는 타당합니다.
- gensim `Word2Vec`: 현재 공식 문서의 생성자에는 `vector_size`, `window`, `min_count`, `sg`, `seed`, `workers` 등이 포함됩니다. 초안 코드의 매개변수명은 현재 API와 맞습니다.
