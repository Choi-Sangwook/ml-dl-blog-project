## 게시 가능

**아래 필수 2건을 수정한 뒤 게시 가능합니다.**

- PDF의 핵심 내용인 퍼셉트론, 논리 게이트, XOR와 다층 신경망, 계층 구조, 순전파·손실·역전파·가중치 업데이트, 활성화 함수가 최종본에 포함되어 있습니다.
- 기존 통합 검토안의 높은 우선순위 사항인 `nn.Linear`의 역할, 배치 행렬 shape, logits·라벨·손실함수 조합, train/validation/test 구분, `train()`·`eval()`·`no_grad()`, 활성화 함수의 과도한 단정 완화가 대부분 올바르게 반영되었습니다.
- AND 학습 예제의 입력·라벨은 모두 `float32`이며, 출력 `(4, 1)` logit과 `nn.BCEWithLogitsLoss()`의 조합이 적절합니다.
- 다중 분류의 `(N, C)` logits, `(N,)` 형태의 `torch.long` 라벨, `nn.CrossEntropyLoss()` 조합도 정확합니다.
- 학습 데이터와 평가 데이터의 역할, 학습된 전처리의 데이터 누수 방지 원칙이 설명되어 있습니다.
- CPU 기준 예제이므로 현재 코드에 device 불일치 문제는 없습니다.
- Markdown 제목, 표, 링크, 코드 fence 및 UTF-8 상태에서 게시를 막을 문제는 확인되지 않았습니다.

## 게시 전 반드시 수정

### 1. 무제한 반복 코드에 실제 종료 조건 추가

최종본의 무작위 가중치 탐색 코드는 `while True`를 사용합니다. 바로 아래에서 위험성을 경고하지만, 독자가 실행하는 실제 코드는 여전히 무기한 반복될 수 있습니다.

- 위치: 최종본 197행 부근
- 현재 코드: `while True:`
- 수정 방향: `for trial in range(max_trials):`처럼 최대 시도 횟수를 두고, 찾지 못했을 때 오류나 안내 메시지를 출력합니다.

### 2. `nn.Linear`를 단일 퍼셉트론으로 부르는 주석 수정

최종본 앞부분에서는 `nn.Linear`가 가중합을 계산하는 선형 계층이며 고전적 퍼셉트론 전체와 같지 않다고 올바르게 설명합니다. 그러나 AND 학습 코드의 주석은 다시 둘을 같은 것으로 표현해 앞 설명과 충돌합니다.

- 위치: 최종본 240행 부근
- 현재 주석: `# 단일 퍼셉트론(선형 계층)`
- 권장 주석: `# 이진 분류용 선형 계층(logit 출력)`

## 선택적으로 개선

- “역전파는 오차의 원인을 추적한다”는 표현을 “손실에 대한 각 파라미터의 기울기를 계산한다”로 바꾸면 인과적 책임을 계산한다는 오해를 줄일 수 있습니다.
- Sigmoid 이후의 활성화 함수 코드 블록은 앞에서 선언한 `x`와 `nn`에 의존합니다. “앞 코드의 변수와 import를 재사용하는 연속 실행 예제”라고 명시하거나 각 블록에 필요한 준비 코드를 포함하면 개별 복사 실행이 쉬워집니다.
- “거의 모든 PyTorch 학습 코드는 이 구조를 따릅니다”는 “기본적인 PyTorch 학습 루프의 골격입니다”로 완화하면 미니배치, 스케줄러, 혼합 정밀도 등을 사용하는 실제 코드까지 포괄하는 단정으로 읽히지 않습니다.
- 역전파·손실함수 조합에 관한 설명이 여러 구역에서 반복되지만 입문 독자를 위한 의도적 반복으로 볼 수 있습니다. 글 길이를 줄여야 할 때만 앞부분의 예고성 설명을 축약하면 됩니다.
